# Modalità d'Esame

L’esame è composto da due parti:

## 1. Prova scritta di ammissione
Una **semplice prova scritta** che testa le conoscenze generali acquisite durante il corso.

⚠️ La prova **non è valida ai fini del voto**, ma è **necessaria per accedere al progetto**.

## 2. Progetto

A partire da una **traccia assegnata**, lo studente dovrà:

1. Progettare lo **schema di un database**
2. **Implementare il database**
3. **Popolarlo con dati di esempio**
4. Scrivere uno **script Python** che esegua le **query richieste**

Il progetto dovrà dimostrare la capacità di:
- progettare un database
- utilizzare query SQL
- interagire con il database tramite Python

# Programma del Corso

## Fondamenti di Database Relazionali (22h)

- Concetti fondamentali dei database relazionali
- Modellazione **ER**
  - entità
  - attributi
  - relazioni
- Cenni su altri modelli di dati
- Linguaggio **SQL**
- Esercitazioni su **SQLite**

---

## Database Non Relazionali (NoSQL) (16h)

- Concetti fondamentali dei database **NoSQL**
- **Key-Value Databases**
- **Document Databases**
- **Graph Databases**
- Esercitazioni pratiche su database NoSQL

---

## Analisi Dati con Python e Pandas (8h)

- Introduzione a **NumPy** e **Pandas**
- **Series** e **DataFrame**
- Operazioni di base
  - selezione
  - slicing
  - filtri
- Importazione dati
  - CSV
  - Excel
  - JSON
- Gestione di **file di grandi dimensioni**
- Interazione con **database**
  - SQL
  - Neo4j
- **Visualizzazione dei dati**
